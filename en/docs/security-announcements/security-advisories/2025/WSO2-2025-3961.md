# Security Advisory WSO2-2025-3961/CVE-2025-3125

<p class="doc-info">Published: 2025-11-05</p>
<p class="doc-info">Version: 1.0.0</p>
<p class="doc-info">Severity: Medium</p>
<p class="doc-info">CVSS Score: 6.7 (CVSS:3.1/AV:N/AC:L/PR:H/UI:N/S:U/C:H/I:H/A:L)</p>
<p class="doc-info">CVE IDs: <a href="https://www.cve.org/CVERecord?id=CVE-2025-3125">CVE-2025-3125</a></p>

### AFFECTED PRODUCTS
- WSO2 API Control Plane 4.5.0
- WSO2 API Manager 4.5.0, 4.4.0, 4.3.0, 4.2.0, 4.1.0, 4.0.0, 3.2.1, 3.2.0
- WSO2 Enterprise Integrator 6.6.0
- WSO2 Identity Server 7.0.0, 6.1.0, 6.0.0, 5.11.0, 5.10.0
- WSO2 Identity Server as Key Manager 5.10.0
- WSO2 Open Banking IAM 2.0.0
- WSO2 Traffic Manager 4.5.0
- WSO2 Universal Gateway 4.5.0


### OVERVIEW
Authenticated arbitrary file upload via CarbonAppUploader admin service endpoint.


### DESCRIPTION
Due to improper validation of user input, an authenticated malicious actor with the required privileges may upload an arbitrary file to a user-controlled location on the server, potentially achieving remote code execution. This permission is by default only available to admin users.


### IMPACT
By leveraging this vulnerability, malicious actors with required privileges may perform remote code execution by uploading a specially crafted malicious file.


### SOLUTION

#### Community Users (Open Source)
Apply the relevant fixes to your product using the public fix(es) provided below.

* [https://github.com/wso2/carbon-commons/pull/516](https://github.com/wso2/carbon-commons/pull/516)


If applying the fix or update is not feasible, migrate to the latest unaffected version of the respective WSO2 product(s).


#### Support Subscription Holders

Update your product to the specified update level or a higher update level to apply the fix.
