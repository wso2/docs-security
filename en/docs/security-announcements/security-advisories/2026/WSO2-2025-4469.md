### AFFECTED PRODUCTS
* WSO2 Identity Server: 7.0.0


### OVERVIEW
Potential Denial-of-Service (DoS) vulnerability via Magic Link authentication.


### DESCRIPTION
This vulnerability impacts only deployments that use the Magic Link authenticator, where multiple invalid authentication requests can cause uncontrolled memory usage growth, resulting in a service unavailability condition.


### IMPACT
In a deployment where Magic Link feature is used, successful exploitation could result in the complete unavailability of the deploymentâ€™s authentication mechanism, preventing legitimate users from accessing the system.


### SOLUTION

#### Community Users (Open Source)
Apply the relevant fixes to your product using the public fix(es) provided below.

* [https://github.com/wso2-extensions/identity-local-auth-magiclink/pull/61](https://github.com/wso2-extensions/identity-local-auth-magiclink/pull/61)

If applying the fix or update is not feasible, migrate to the latest unaffected version of the respective WSO2 product(s).
