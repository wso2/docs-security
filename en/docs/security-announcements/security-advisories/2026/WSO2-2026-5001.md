### AFFECTED PRODUCTS
* WSO2 API Control Plane: 4.6.0, 4.5.0
* WSO2 API Manager: 4.6.0, 4.5.0, 4.4.0, 4.3.0, 4.2.0, 4.1.0, 3.2.1, 3.2.0
* WSO2 Traffic Manager: 4.6.0, 4.5.0
* WSO2 Universal Gateway: 4.6.0, 4.5.0


### OVERVIEW
XML External Entity (XXE) injection vulnerability in XMLSchemaValidator Mediator.


### DESCRIPTION
The XML and schema validation functionalities within the XMLSchemaValidator Mediator process XML input as part of validation flows. Under certain conditions, XML parsing allows the resolution of external entities. This behavior can occur when handling user-supplied XML content during validation operations. Exploitation requires access to the relevant mediator flows with sufficient privileges to supply crafted XML payloads.


### IMPACT
Successful exploitation may allow a highly privileged actor to read files accessible within the server hosting the affected product. In addition, it may be possible to trigger outbound requests to unintended internal or external locations, depending on the server environment and network configuration. Additionally, specially crafted XML payloads may lead to excessive resource consumption during parsing, which could impact the availability.


### SOLUTION

#### Community Users (Open Source)
Apply the relevant fixes to your product using the public fix(es) provided below.

* [https://github.com/wso2/carbon-apimgt/pull/13663](https://github.com/wso2/carbon-apimgt/pull/13663)

If applying the fix or update is not feasible, migrate to the latest unaffected version of the respective WSO2 product(s).

Update your product to the specified update level, or to a higher update level, to mitigate the identified vulnerability.
