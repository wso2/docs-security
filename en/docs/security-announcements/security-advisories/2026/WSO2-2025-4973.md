* WSO2 Traffic Manager: 4.6.0, 4.5.0
* WSO2 Universal Gateway: 4.6.0, 4.5.0


### OVERVIEW
Potential account takeover via Conditional Authentication (Adaptive Authentication).


### DESCRIPTION
When a Conditional Authentication (Adaptive Authentication) script follows a specific multi-step pattern and certain conditions are met, a malicious actor can gain access to user accounts by completing only the initial authentication steps.</span></p><p><br></p><p><span style="color: rgb(0, 0, 0); background-color: transparent;">Affected WSO2 Products that use Conditional Authentication are impacted if the following conditions are met.</span></p><p><br></p><p><strong style="color: rgb(0, 0, 0); background-color: transparent;">Conditions:</strong></p><p><br></p><p><span style="background-color: transparent;">1. Application login flow contains one of the following authenticators as a secondary step</span></p><p><br></p><p><br></p><ul><li><span style="background-color: transparent;">Authenticators provided by WSO2 Identity Server 7.0.0, 7.1.0, and 7.2.0</span></li><li class="ql-indent-1"><span style="background-color: transparent;">Passkey </span>(FIDO2) <span style="background-color: transparent;">Authentication</span></li><li class="ql-indent-1"><span style="background-color: transparent;">Push Notification Based Authentication</span></li><li class="ql-indent-1"><span style="background-color: transparent;">iProov Authentication</span></li><li><span style="background-color: transparent;">Authenticators provided by WSO2 Identity Server 5.11.0 update level between 399 to 431 (both versions included)</span></li><li class="ql-indent-1"><span style="background-color: transparent;">Passkey (FIDO2) Authentication </span></li></ul><p><br></p><p><br></p><p><br></p><ul><li><span style="background-color: transparent;">Any custom authenticators used in the impacted product version</span></li><li class="ql-indent-1"><span style="background-color: transparent;">A custom authenticator with logic to return any other status than SUCCESS_COMPLETE or INCOMPLETE&nbsp;</span></li></ul><p><br></p><p><span style="background-color: transparent;">2. The Conditional Authentication script meets both of the following conditions:</span></p><p><br></p><p><br></p><ul><li><span style="background-color: transparent;">The executeStep function for one of the above authenticators, and has defined at least one event callback function, such as onFail,onUserAbort, onFallback, or onSuccess.</span></li><li><span style="background-color: transparent;">The same authentication step is executed again in the script.</span></li></ul><p><br></p><p><span style="background-color: transparent;">3. The targeted user should have at least one of the aforementioned impacted authenticators available for enrollment.</span></p><p><br></p><p><span style="background-color: transparent;">4. The malicious actor must complete any prior authentication steps before the vulnerable step.


### IMPACT
Successful exploitation of this vulnerability would allow a malicious actor to gain access to a targeted account.


### SOLUTION

#### Community Users (Open Source)
Apply the relevant fixes to your product using the public fix(es) provided below.

* [https://github.com/wso2/carbon-identity-framework/pull/7660](https://github.com/wso2/carbon-identity-framework/pull/7660)
* [https://github.com/wso2/carbon-identity-framework/pull/7663](https://github.com/wso2/carbon-identity-framework/pull/7663)

If applying the fix or update is not feasible, migrate to the latest unaffected version of the respective WSO2 product(s).


#### Support Subscription Holders

Update your product to the specified update level, or to a higher update level, to mitigate the identified vulnerability.
