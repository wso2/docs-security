### AFFECTED PRODUCTS
* WSO2 API Control Plane: 4.6.0, 4.5.0
* WSO2 API Manager: 4.6.0, 4.5.0, 4.4.0, 4.3.0, 4.2.0, 4.1.0, 4.0.0, 3.2.1, 3.2.0, 3.1.0
* WSO2 Identity Server as Key Manager: 5.10.0
* WSO2 Identity Server: 7.2.0, 7.1.0, 7.0.0, 6.1.0, 6.0.0, 5.11.0, 5.10.0
* WSO2 Open Banking AM: 2.0.0
* WSO2 Open Banking IAM: 2.0.0
* WSO2 Traffic Manager: 4.6.0, 4.5.0
* WSO2 Universal Gateway: 4.6.0, 4.5.0


### OVERVIEW
Arbitrary Remote Code Execution vulnerability with admin permission.


### DESCRIPTION
Improper input validation allows a malicious actor with administrative privileges and Carbon Console access to inject and execute arbitrary code remotely through specific administrative operations.


### IMPACT
Improper input validation allows a malicious actor with admin privileges and carbon console access to execute remote arbitrary code through specific administrative operations.


### SOLUTION

#### Community Users (Open Source)
Apply the relevant fixes to your product using the public fix(es) provided below.

* [https://github.com/wso2/product-is/pull/26281](https://github.com/wso2/product-is/pull/26281)


If applying the fix or update is not feasible, migrate to the latest unaffected version of the respective WSO2 product(s).


#### Support Subscription Holders

Update your product to the specified update level, or to a higher update level, to mitigate the identified vulnerability.
