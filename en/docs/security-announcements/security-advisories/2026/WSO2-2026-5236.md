### AFFECTED PRODUCTS
* WSO2 API Control Plane: 4.6.0, 4.5.0
* WSO2 API Manager: 4.6.0, 4.5.0, 4.4.0, 4.3.0, 4.2.0, 4.1.0, 4.0.0, 3.2.1, 3.2.0
* WSO2 Traffic Manager: 4.6.0, 4.5.0
* WSO2 Universal Gateway: 4.6.0, 4.5.0


### OVERVIEW
Denial of service vulnerability.


### DESCRIPTION
An unauthenticated remote attacker can inject malicious JSON payloads into the Throttling events, causing a persistent Denial of Service (DoS) that impacts the API Gateway.


### IMPACT
Successful exploitation of this vulnerability could allow an unauthenticated malicious actor to disrupt the API Gateway by triggering a persistent Denial of Service condition, preventing legitimate API traffic from being processed and impacting complete service availability.


### SOLUTION

#### Community Users (Open Source)
Apply the relevant fixes to your product using the public fix(es) provided below.

* [https://github.com/wso2/product-apim/pull/14217](https://github.com/wso2/product-apim/pull/14217)

If applying the fix or update is not feasible, migrate to the latest unaffected version of the respective WSO2 product(s).


#### Support Subscription Holders

Update your product to the specified update level, or to a higher update level, to mitigate the identified vulnerability.
