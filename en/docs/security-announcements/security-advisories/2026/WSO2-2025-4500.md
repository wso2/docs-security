### AFFECTED PRODUCTS
* WSO2 API Manager : 4.2.0, 4.1.0, 4.0.0, 3.2.1, 3.2.0


### OVERVIEW
Authenticated Stored Cross-Site Scripting (XSS).


### DESCRIPTION
Due to improper validation, a user with the Publisher role can exploit a stored Cross-Site Scripting (XSS) vulnerability by uploading an SVG file to the Publisher portal.


### IMPACT
By leveraging the XSS attack, a malicious actor can make the browser get redirected to an arbitrary website, make changes in the UI of the web page, retrieve information from the browser, or cause harm otherwise. However, since all the session-related sensitive cookies are set with the httpOnly flag and protected, session hijacking or similar attacks would not be possible.


### SOLUTION



#### Community Users (Open Source)
Apply the relevant fixes to your product using the public fix(es) provided below.

* [https://github.com/wso2/api-manager/issues/1280](https://github.com/wso2/api-manager/issues/1280)

If applying the fix or update is not feasible, migrate to the latest unaffected version of the respective WSO2 product(s).

Update your product to the specified update level or a higher update level to apply the fix.
