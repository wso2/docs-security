### OVERVIEW
Potential availability impact on tenants


### DESCRIPTION
When the deployment is configured with a MySQL userstore the application users cannot access their applications due to the cache entries getting created against the incorrect tenant domain. This happens due to some special characters in MySQL are treated as similar alphanumeric characters.


### IMPACT
Due to the identified vulnerability the legitimate users of the tenant will not be able to access the application.


### SOLUTION

#### Community Users (Open Source)
Apply the relevant fixes to your product using the public fix(es) provided below.

* [https://github.com/wso2-extensions/identity-carbon-auth-rest/pull/344/](https://github.com/wso2/carbon-kernel/pull/4430)

If applying the fix or update is not feasible, migrate to the latest unaffected version of the respective WSO2 product(s).


#### Support Subscription Holders

Update your product to the specified update level, or to a higher update level, to mitigate the identified vulnerability.
