### AFFECTED PRODUCTS
* WSO2 API Control Plane: 4.5.0
* WSO2 API Manager: 4.5.0, 4.4.0, 4.3.0, 4.2.0, 4.1.0, 3.2.1, 3.2.0
* WSO2 Universal Gateway: 4.5.0


### OVERVIEW
Refresh token misuse due to validation flaw in the ForgeRock client.


### DESCRIPTION
Due to improper validation of the refresh token, it can be incorrectly treated as an access token, allowing it to be used to consume protected resources when refresh tokens are issued by ForgeRock identity provider.


### IMPACT
If a refresh token is exposed or stolen by a malicious actor, it may be used to access protected resources. Although the standard flow requires both the refresh token and valid client credentials (which are not publicly available) to obtain an access token for consuming the protected resources, this requirement can be bypassed through the identified vulnerability.


### SOLUTION

#### Community Users (Open Source)
Apply the relevant fixes to your product using the public fix(es) provided below.

* [https://github.com/wso2-extensions/apim-km-forgerock/pull/13](https://github.com/wso2-extensions/apim-km-forgerock/pull/13)

If applying the fix or update is not feasible, migrate to the latest unaffected version of the respective WSO2 product(s).

#### Support Subscription Holders

Update your product to the specified update level, or to a higher update level, to mitigate the identified vulnerability.
