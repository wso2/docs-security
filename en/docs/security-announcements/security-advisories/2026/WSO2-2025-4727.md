### OVERVIEW
Improper token revocation on session-bound token flow.


### DESCRIPTION
Due to improper implementation of the session-bound token feature [^1], tokens associated with user sessions are not revoked when the corresponding user session is terminated, unless another SSO request is initiated. This feature is disabled by default in WSO2 products. However, it can be enabled from the console by navigating to the application protocol configuration section, selecting the token binding type as “SSO Session”, and enabling the “Revoke token upon user logout” option.


### IMPACT
This vulnerability exists when token binding validation is enabled. As a result, a malicious actor possessing a valid session-bound token could exploit this condition to maintain unauthorized access to applications and resources even after the associated user session has ended, potentially leading to the exposure of sensitive user data.


### SOLUTION

#### Community Users (Open Source)
Apply the relevant fixes to your product using the public fix(es) provided below.

* [https://github.com/wso2/product-apim/pull/13967](https://github.com/wso2/product-apim/pull/13967)
* [https://github.com/wso2-extensions/identity-inbound-auth-oauth/pull/3053](https://github.com/wso2-extensions/identity-inbound-auth-oauth/pull/3053)
* [https://github.com/wso2/carbon-identity-framework/pull/7757](https://github.com/wso2/carbon-identity-framework/pull/7757)

If applying the fix or update is not feasible, migrate to the latest unaffected version of the respective WSO2 product(s).


#### Support Subscription Holders

Update your product to the specified update level, or to a higher update level, to mitigate the identified vulnerability.
