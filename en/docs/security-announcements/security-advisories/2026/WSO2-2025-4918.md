### AFFECTED PRODUCTS
* WSO2 API Control Plane: 4.6.0, 4.5.0
* WSO2 API Manager: 4.6.0, 4.5.0, 4.4.0, 4.3.0, 4.2.0, 4.1.0
* WSO2 Traffic Manager: 4.6.0, 4.5.0
* WSO2 Universal Gateway: 4.6.0, 4.5.0


### OVERVIEW
Cross-tenant access control bypass in Publisher REST APIs on multi-tenant deployments.


### DESCRIPTION
In multi-tenant deployments, tenant isolation is not correctly enforced for Publisher REST APIs. A user in one tenant with sufficient privileges to invoke the Publisher REST API can perform operations impacting other tenants, bypassing tenant-level restrictions.


### IMPACT
The vulnerability affects only multi-tenant deployments where it allows a privileged user to perform publisher operations such as exposing or modifying API Metadata in another tenant environment.


### SOLUTION

#### Community Users (Open Source)
Apply the relevant fixes to your product using the public fix(es) provided below.

* [https://github.com/wso2/carbon-apimgt/pull/13546](https://github.com/wso2/carbon-apimgt/pull/13546)

If applying the fix or update is not feasible, migrate to the latest unaffected version of the respective WSO2 product(s).

Update your product to the specified update level, or to a higher update level, to mitigate the identified vulnerability.
