### AFFECTED PRODUCTS
* WSO2 API Control Plane: 4.5.0
* WSO2 API Manager: 4.5.0, 4.4.0, 4.3.0, 4.2.0, 4.1.0
* WSO2 Traffic Manager: 4.5.0
* WSO2 Universal Gateway: 4.5.0


### OVERVIEW
Potential Header injection.


### DESCRIPTION
In Webhook API invocations, a malicious actor may add or overwrite HTTP response headers by manipulating HTTP request headers, resulting in HTTP header injection vulnerability.


### IMPACT
By leveraging the HTTP response header injection vulnerability, a malicious actor may inject or overwrite any response header, including cookie values, adversely impact browser caching behaviors, alter security-related HTTP headers, or cause harm otherwise.

### SOLUTION

#### Community Users (Open Source)
Apply the relevant fixes to your product using the public fix(es) provided below.

* [https://github.com/wso2/carbon-apimgt/pull/13216](https://github.com/wso2/carbon-apimgt/pull/13216)
* [https://github.com/wso2/product-apim/pull/13783](https://github.com/wso2/product-apim/pull/13783)
* [https://github.com/wso2/product-apim/pull/13796](https://github.com/wso2/product-apim/pull/13796)

If applying the fix or update is not feasible, migrate to the latest unaffected version of the respective WSO2 product(s).

#### Support Subscription Holders

Update your product to the specified update level, or to a higher update level, to mitigate the identified vulnerability.
