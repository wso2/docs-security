# Security Advisory WSO2-2023-2606

<p class="doc-info">Published: 2026-05-03</p>

### AFFECTED PRODUCTS
* WSO2 API Manager: 4.2.0, 4.1.0, 4.0.0, 3.2.0, 3.1.0, 3.0.0
* WSO2 Identity Server as Key Manager: 5.10.0, 5.9.0
* WSO2 Identity Server: 6.1.0, 6.0.0, 5.11.0, 5.10.0, 5.9.0
* WSO2 Open Banking AM: 2.0.0
* WSO2 Open Banking IAM: 2.0.0


### OVERVIEW
HTTP TRACE method listed as an allowed HTTP method.


### DESCRIPTION
When sending requests to API endpoints using the HTTP OPTIONS method the HTTP TRACE method is listed as an allowed HTTP method in the response even though the endpoints do not support HTTP TRACE. This update will remove the HTTP TRACE method from the response to a HTTP OPTION request. Kindly note that we have done necessary analysis and confirm all the endpoints are not allowed to use HTTP TRACE method. Therefore we are delivering this update as an improvement to avoid similar queries in the future including detections from automated scanners.


### IMPACT
There is no security impact because all the API endpoints do not respond to theHTTP TRACE method in a way that could cause a security impact. However we are delivering this update as an improvement.


### SOLUTION

#### Community Users (Open Source)
Migrate to the latest unaffected version of the respective WSO2 product(s).


#### Support Subscription Holders

Update your product to the specified update level, or to a higher update level, to mitigate the identified vulnerability.
