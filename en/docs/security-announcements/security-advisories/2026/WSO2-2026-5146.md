### AFFECTED PRODUCTS
* WSO2 API Control Plane: 4.6.0, 4.5.0
* WSO2 API Manager: 4.6.0, 4.5.0, 4.4.0
* WSO2 Traffic Manager: 4.6.0, 4.5.0
* WSO2 Universal Gateway: 4.6.0, 4.5.0


### OVERVIEW
Authenticated arbitrary file upload via a System REST API.


### DESCRIPTION
A malicious actor with administrative privileges can upload an arbitrary file to a user-controlled location within the deployment via a system REST API. Successful uploads may lead to remote code execution. Exploitation requires authenticated access with the publisher privileges.


### IMPACT
Successful exploitation may allow an authenticated publisher to upload files to server-accessible locations. Depending on the deployment environment and how uploaded files are handled, this could lead to execution of uploaded content.


### SOLUTION

#### Community Users (Open Source)
Apply the relevant fixes to your product using the public fix(es) provided below.

* [https://github.com/wso2/carbon-apimgt/pull/13758](https://github.com/wso2/carbon-apimgt/pull/13758)

If applying the fix or update is not feasible, migrate to the latest unaffected version of the respective WSO2 product(s).


#### Support Subscription Holders

Update your product to the specified update level, or to a higher update level, to mitigate the identified vulnerability.
